# Anytype Self-Hosting Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy a self-hosted Anytype sync network (`any-sync`) on the home-ops Kubernetes cluster, accessible remotely via Cloudflare WARP private network routing.

**Architecture:** Custom Helm chart at `charts/anytype/` containing MongoDB, redis-stack-server, and all four any-sync services. A one-time manual config generation step (run locally before first deploy) produces crypto keys and topology configs stored in Bitwarden, pulled via ExternalSecrets. All four sync services share LoadBalancer IP `192.168.48.29` on different ports (TCP + QUIC/UDP each).

**Tech Stack:** Helm, Kubernetes, Cilium LB IPAM, ExternalSecrets / Bitwarden Secrets Manager, any-sync-network CLI tool, MongoDB 7.0, redis-stack-server 7.2, any-sync v0.9–0.11.

## Global Constraints

- Namespace: `anytype`
- LB IP: `192.168.48.29` — verify it is free before enabling the app (currently assigned IPs: .20 envoy-external, .21 envoy-internal, .22 jellyfin, .27 HA audio, .28 vintagestory)
- Storage class: `ceph-block` for all PVCs
- ExternalSecret `remoteRef.key` values are Bitwarden secret **UUIDs** (not names) — replace all `REPLACE-WITH-UUID` placeholders after creating the Bitwarden secrets
- Image tags must be pinned (they are pinned in `values.yaml`; Renovate manages updates)
- Never commit secret values — gitleaks pre-commit hook enforces this
- After every template change: `helm template anytype . -f values.yaml` from `charts/anytype/` must succeed before committing
- After every change to `cluster/apps/default/anytype/`: run `helm dependency update` then `helm template anytype . -f values.yaml` from that directory
- Branch: work on `feat/anytype`

### Task 1: One-Time Config Generation (Manual — Run Before Any K8s Steps)

This task has no K8s files. It produces the Bitwarden secrets that all later tasks depend on.

**Files:** None (manual steps only)

- [ ] **Step 1: Install the any-sync-network tool**

```bash
go install github.com/anyproto/any-sync-tools/any-sync-network@latest
```

Expected: `$GOPATH/bin/any-sync-network` binary present.

- [ ] **Step 2: Run interactive config generation**

```bash
any-sync-network create
```

When prompted, enter the following values:
- External host for **all** services: `192.168.48.29`
- Coordinator TCP port: `1004`, QUIC port: `1014`
- Consensus node TCP port: `1006`, QUIC port: `1016`
- Sync node TCP port: `1001`, QUIC port: `1011`
- File node TCP port: `1005`, QUIC port: `1015`

This produces: `coordinator.yml`, `consensus.yml`, `sync_1.yml`, `file_1.yml`, `network.yml`, `heart.yml`.

- [ ] **Step 3: Augment coordinator.yml with MongoDB connection**

Open `coordinator.yml` and set the MongoDB field to:
```
mongodb://mongo-1.anytype.svc.cluster.local:27001/?replicaSet=rs0
```
(The exact field name is `mongo.connect` or similar — check the generated file.)

- [ ] **Step 4: Augment consensus.yml with MongoDB connection**

Same field as coordinator.yml — point to the same MongoDB URI:
```
mongodb://mongo-1.anytype.svc.cluster.local:27001/?replicaSet=rs0
```

- [ ] **Step 5: Augment file_1.yml with Redis URL and QNAP S3 config**

In `file_1.yml`, set:
- Redis URL: `redis://redis.anytype.svc.cluster.local:6379`
- S3 endpoint: your QNAP QuObjects endpoint URL (e.g. `https://qnap.local:9000`)
- S3 bucket: your chosen bucket name (create it in QuObjects first)
- S3 region: `us-east-1` (required by SDK even for non-AWS S3)
- S3 profile: `default`

- [ ] **Step 6: Create Bitwarden secrets and note the UUIDs**

In Bitwarden Secrets Manager, create one secret per file. Store the **entire file content** as the secret value. Record each UUID for use in Task 5.

| Secret name | Content |
|---|---|
| `anytype-coordinator-config` | contents of `coordinator.yml` |
| `anytype-network-config` | contents of `network.yml` |
| `anytype-consensus-config` | contents of `consensus.yml` |
| `anytype-syncnode-config` | contents of `sync_1.yml` |
| `anytype-filenode-config` | contents of `file_1.yml` |
| `anytype-aws-access-key-id` | QNAP QuObjects access key ID (string) |
| `anytype-aws-secret-access-key` | QNAP QuObjects secret access key (string) |

- [ ] **Step 7: Save heart.yml to Bitwarden for backup**

Store `heart.yml` as `anytype-heart-config` in Bitwarden for safekeeping. Also keep a local copy — you will paste it into the Anytype app in Task 10.

- [ ] **Step 1: Create charts/anytype/templates/mongodb-statefulset.yaml**

```yaml
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: mongo-1
  namespace: {{ .Values.namespace }}
spec:
  serviceName: mongo-1
  replicas: 1
  selector:
    matchLabels:
      app: mongo-1
  template:
    metadata:
      labels:
        app: mongo-1
    spec:
      containers:
        - name: mongo
          image: {{ .Values.mongodb.image.repository }}:{{ .Values.mongodb.image.tag }}
          args:
            - --replSet
            - {{ .Values.mongodb.replicaSetName }}
            - --port
            - {{ .Values.mongodb.port | quote }}
            - --bind_ip_all
          ports:
            - containerPort: {{ .Values.mongodb.port }}
          lifecycle:
            postStart:
              exec:
                command:
                  - /bin/sh
                  - -c
                  - |
                    until mongosh --port {{ .Values.mongodb.port }} --quiet \
                      --eval "db.adminCommand('ping').ok" 2>/dev/null | grep -q 1; do
                      sleep 2
                    done
                    mongosh --port {{ .Values.mongodb.port }} --quiet --eval \
                      "try{rs.initiate({_id:'{{ .Values.mongodb.replicaSetName }}',members:[{_id:0,host:'mongo-1.{{ .Values.namespace }}.svc.cluster.local:{{ .Values.mongodb.port }}'}]})}catch(e){}" \
                      || true
          readinessProbe:
            exec:
              command:
                - /bin/sh
                - -c
                - |
                  mongosh --port {{ .Values.mongodb.port }} --quiet \
                    --eval "rs.status().ok" 2>/dev/null | grep -q 1
            initialDelaySeconds: 30
            periodSeconds: 10
            failureThreshold: 6
          livenessProbe:
            exec:
              command:
                - mongosh
                - --port
                - {{ .Values.mongodb.port | quote }}
                - --quiet
                - --eval
                - "db.adminCommand('ping')"
            initialDelaySeconds: 30
            periodSeconds: 15
          volumeMounts:
            - name: data
              mountPath: /data/db
  volumeClaimTemplates:
    - metadata:
        name: data
      spec:
        accessModes: [ReadWriteOnce]
        storageClassName: {{ .Values.mongodb.storage.storageClass }}
        resources:
          requests:
            storage: {{ .Values.mongodb.storage.size }}
```
