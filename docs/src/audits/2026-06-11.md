## Critical & Warning Issues

### 🔴 CRITICAL

**`KubeProxyDown` — firing since 2026-04-23**
This alert has been firing for ~7 weeks. On a Talos + Cilium cluster this is **expected** — Cilium replaces kube-proxy and no kube-proxy DaemonSet runs. The alert rule should be silenced or removed from Prometheus rules. Not a real problem but creates alert fatigue.

**`hermes-agent` — crash loop + CPU spike**
- Pod restarted 3× in the last ~100 minutes (exit code 1)
- Root cause: `ERROR gateway.run: Another gateway instance (PID 155) started during our startup. Exiting to avoid double-running.` — the s6-supervise init system is racing with itself on startup and the load average reaches 199 during the crash
- CPU jumped from **4m** (7d ago) to **514m** now (over its 200m request, within 1500m limit)
- Likely a stuck/zombie gateway process that isn't cleaned up on container restart — try `kubectl -n hermes-agent delete pod hermes-agent-bbfd6bc7d-ncwjd` to clear, then investigate the s6-supervise config
