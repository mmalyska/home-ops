# Talos configuration as code

Folder contains cluster definition using `talosctl` + `envsubst`.

## Structure

```text
templates/
  controlplane.yaml         # base config for control plane nodes (${TALHELPER_*} placeholders)
  worker.yaml               # base config for worker nodes
  talosctl-secrets.yaml     # maps TALHELPER_* vars to talosctl secrets bundle format
  extension-nut-client.yaml # ExtensionServiceConfig for nut-client UPS monitoring
nodes/
  mc1.yaml, mc2.yaml, mc3.yaml, nv1.yaml   # per-node strategic merge patches
nodes.yaml                  # source of truth: name, IP, type for each node
clusterconfig/              # gitignored; generated by task talos:generate
```

## Usage

Generate all node configs and talosconfig:

```bash
task talos:generate
```

Apply config to a node (no reboot):

```bash
task talos:apply N=mc1
```

Apply config with reboot:

```bash
task talos:apply:restart N=mc1
```

Upgrade Talos on all nodes:

```bash
task talos:upgrade:all
```

Bootstrap a fresh cluster (first-time only):

```bash
talosctl --nodes 192.168.48.2 apply-config --insecure -f provision/talos/clusterconfig/home-mc1.yaml
kubectl get csr -o name | grep "certificates.k8s.io" | xargs kubectl certificate approve
```

Apply CNI and ArgoCD after bootstrap:

```bash
helm dependencies update cluster/apps/core/cilium
helm template -n kube-system cluster/apps/core/cilium | kubectl apply -f -
kustomize build cluster/apps/core/argocd | argocd-secret-replacer sops -f cluster/apps/core/argocd/secret.sec.yaml | kubectl apply -f -
```

## Adding a new node

1. Add entry to `nodes.yaml` (name, ip, type)
2. Create `nodes/<name>.yaml` with hostname, interface, IP (and VIP if controlplane)
3. Run `task talos:generate`
4. For non-standard config (different disk, sysctls), add extra merge patch keys to `nodes/<name>.yaml`

## Secrets

All secrets use `TALHELPER_*` environment variables populated from Bitwarden via `.envrc`.
See `templates/talosctl-secrets.yaml` for the mapping.
